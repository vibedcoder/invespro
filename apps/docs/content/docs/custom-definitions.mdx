---
title: Custom Definitions
description: Customize questions, scoring, profiles, bands, overrides, and allocations.
---

Custom definitions are the controlled customization path. They let a team adapt
the model without writing an arbitrary decision graph from scratch.

## What can be customized

- Definition ID and version.
- Question IDs, labels, types, and options.
- Scoring rules and weights.
- Profile IDs and labels.
- Score bands.
- Asset classes and allocation percentages.
- Rule-based overrides.
- Informational questions that do not affect score.

## Example

```ts
import { RiskProfilerEngine } from "@vibedcoder/invespro-core";
import type { RiskProfileDefinition } from "@vibedcoder/invespro-types";

const definition: RiskProfileDefinition = {
  schemaVersion: "1.0",
  id: "exampleDefinition",
  name: "Example Definition",
  version: "0.1.0",
  currency: "AUD",
  questions: [],
  scoring: [],
  profiles: [],
  scoreBands: [],
  assetClasses: [],
  allocations: {},
  overrides: [],
};

const engine = new RiskProfilerEngine({ definition });
```

The real definition must satisfy the schema and the graph contract expected by
the engine. Use the demo validator or the shared schemas before deploying a
custom model.

## Expert JDM mode

Arbitrary JDM should remain an expert mode. It is powerful, but it must conform
to the Invespro graph contract so the result shape still works with the core,
REST adapter, CLI, and demo.
